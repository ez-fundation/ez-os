# EZ-OS: O Protocolo de Nascimento (The Birth Protocol)

> *"Eu não nasço pronto. Eu nasço atento."*

Para garantir a simbiose, o EZ-OS utiliza um processo de **Instalação Flash & Play** e um motor de **Evolução Adaptativa (ART-Lite)**.

## 1. Instalação: Flash & Play (R36S Style)
Inspirado em consoles retro portáteis:
1.  **Baixar**: O usuário baixa o `ez-os-core.zip` (ou clona o repo).
2.  **Copiar**: Joga os arquivos na raiz do dispositivo (PC, Raspberry, SD Card).
3.  **Boot**: Roda `python boot.py`.
    - *Zero Config*: Não é necessário instalar dependências complexas. O Python padrão basta.

## 2. O Ritual de Nascimento (ART-Lite)
Em vez de deletar pastas (que é destrutivo), o sistema usa **Adaptive Resonance Theory** para *reforçar* caminhos neurais.

### O Teste de Ressonância
No primeiro boot, o sistema apresenta cenários lúdicos na TUI:

> *"Você encontra um arquivo corrompido antigo. O que faz?"*
> [A] Tento recuperar cada byte. (Reforça: **Guardian**)
> [B] Apago e abro espaço para o novo. (Reforça: **Idea_Spark**)
> [C] Guardo o fragmento como arte glitch. (Reforça: **Jester**)

### Mecânica (Plasticidade vs Estabilidade)
- **Não há exclusão**: As outras personalidades não morrem, apenas ficam "adormecidas" (baixo peso sináptico).
- **Evolução**: Se o usuário mudar de comportamento meses depois, o sistema *percebe* a nova ressonância e adapta o arquétipo suavemente.
- **Memória**: Todas as escolhas são salvas no `memory.db` como vetores de afinidade.
